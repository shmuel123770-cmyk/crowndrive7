# CrownDrive V2 — דוח תיקון ובדיקות

## תוצאה ארכיטקטונית
- הקוד הישן של יותר מ־8,000 שורות הוחלף בפרויקט מודולרי של כ־1,400 שורות בקוד הליבה והשרת.
- 10 מודולי JavaScript בדפדפן.
- 16 Netlify Functions בצד השרת.
- מאזין `onAuthStateChanged` אחד בלבד.
- אפס כתיבות ישירות ל־Realtime Database מתוך קוד הדפדפן.
- אין `RT_REF.set` ואין החלפה של כל מצב האתר.
- אין Netlify memory DB.
- אין מזהי DOM כפולים בקובץ הראשי.

## אבטחה
- כל כתיבה למסד חסומה ב־Firebase Rules לדפדפן ועוברת דרך Firebase Admin ב־Netlify Functions.
- כל Function מאמתת Firebase ID token ובודקת תפקיד/בעלות/צד להזמנה.
- Storage חסום לקריאה ולכתיבה ישירה מהדפדפן.
- קבצים מועלים ונקראים באמצעות Signed URLs קצרי־חיים בלבד.
- SVG וקבצי MIME לא מאושרים נחסמים; מותרות תמונות JPEG/PNG/WebP/HEIC וסרטוני MP4/MOV/WebM בלבד.
- בעל רכב מקבל גישה למסמכי שוכר רק בזמן הזמנה ממתינה, מאושרת או פעילה. לאחר סיום, דחייה או ביטול, הגישה נדחית.
- כתובת מלאה נמסרת לשוכר רק לאחר אישור ההזמנה.
- תוכן שמגיע ממשתמשים עובר escaping לפני הצגה ב־HTML.
- נוספו CSP, Permissions Policy ו־security headers.
- פעולות רגישות נרשמות ב־`auditLogs`.

## זרימות שנכללו
- הרשמה, כניסה, יציאה ושחזור session.
- אימות מייל אוטומטי לאחר חזרה/רענון.
- רישיון קדמי/אחורי וסלפי מתוך הפרופיל.
- מצלמה אמיתית דרך `getUserMedia` וכפתור גלריה נפרד.
- אישור/דחיית אימות על ידי מנהל.
- תאריך לידה ואכיפת גיל מינימלי לכל רכב.
- תמונה אוטומטית לרכב דרך Wikimedia Commons, עם אפשרות URL ידנית.
- כתובת פרטית, מסירה ועלות מסירה.
- מניעת הזמנות חופפות.
- הודעות לפי הזמנה.
- הוכחת תשלום עם סכום ותמונה.
- תיעוד לפני נסיעה והחזרה: סרטון, דלק, מיילים והערות.
- בעל רכב יכול לסיים הזמנה רק לאחר שני שלבי התיעוד.
- דירוג רכב, שוכר ובעל רכב רק לאחר סיום הזמנה וללא דירוג עצמי.
- כלי העברת legacy שאינו מוחק נתונים קיימים ומנסה להתאים משתמשים לפי UID או מייל Firebase Auth.

## בדיקות שבוצעו
- `node --check` לכל מודולי הדפדפן ולכל Netlify Functions.
- בדיקת JSON תקין ל־Firebase Rules.
- בדיקה סטטית שמכשילה build אם:
  - מספר מאזיני Auth אינו 1.
  - חוזרת כתיבת מסד ישירה מהדפדפן.
  - חוזרת כתיבת state מלאה.
  - קיימים מזהי HTML כפולים.
  - חסרה Function נדרשת.
  - Storage אינו חסום לגישה ישירה.
- 5 unit tests: escaping, סיסמה, סטטוסים, אימות וכוכבים.
- `npm run build` עבר בהצלחה; בדיקת Chromium סומנה כ־skipped כי סביבת ה־sandbox כאן אינה מאפשרת ל־Chromium לעלות. היא נשארה בפרויקט ותרוץ בסביבה שתומכת בכך.
- ZIP integrity נבדק לאחר האריזה.

## מה עדיין חייב בדיקת Staging
בלי פריסה ומשתני Netlify אמיתיים אין אפשרות לאמת כאן end-to-end מול החשבונות הפרטיים שלך. לפני מעבר לחי חובה לבדוק את כל הרשימה ב־`SETUP_V2_HE.md`, כולל ניסיון גישה לא מורשה שחייב להחזיר 403.
