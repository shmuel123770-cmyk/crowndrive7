# CrownDrive V2 — דוח תיקון ובדיקות

## מה תוקן ארכיטקטונית
- נבנה קוד חדש ומודולרי במקום `index.html` של יותר מ-8,000 שורות.
- מאזין `onAuthStateChanged` יחיד בלבד (`js/auth.js`).
- אין `RT_REF.set` ואין כתיבה של כל מצב האתר בבת אחת.
- אין Netlify memory DB.
- הנתונים מפוצלים ל-`users`, `verificationStatus`, `cars`, `bookings`, `messages`, `payments`, `privateUserDocuments`, `privateCarDetails`, `ratings`.
- כל עדכון הוא נקודתי.
- נתוני legacy הם קריאה בלבד למנהל לצורך העתקה; כלי ההעברה אינו מוחק את המקור.

## אבטחה
- Rules לפי UID, תפקיד, בעל הרכב ושוכר.
- סטטוס אימות נשמר בנתיב נפרד שרק מנהל יכול לכתוב אליו.
- שינויי סטטוס הזמנה ותיעוד מסירה עוברים דרך Netlify Function שמאמתת Firebase ID token ומעברי סטטוס.
- Storage חסום לחלוטין לגישה ישירה מהדפדפן.
- העלאה וצפייה בקבצים נעשות דרך Signed URLs קצרים לאחר בדיקת הרשאה בשרת.
- בעל רכב יכול לקבל קישורים זמניים למסמכי שוכר רק בזמן הזמנה ממתינה/מאושרת/פעילה. לאחר סיום, דחייה או ביטול, ההרשאה נכשלת.
- כתובת מלאה מוחזרת לשוכר רק לאחר אישור ההזמנה.
- נוספו CSP ו-Headers בסיסיים.
- תוכן משתמש עובר `esc()` לפני הכנסה ל-HTML בממשקים החדשים.

## יכולות שנכללו
- הרשמה וכניסה עם מייל/סיסמה; דרישות סיסמה ברורות.
- אימות מייל.
- רישיון קדמי/אחורי וסלפי מתוך פרופיל, עם גלריה או מצלמה.
- אישור אימות על ידי מנהל.
- חסימת הזמנה עד השלמת אימות ואישור מנהל.
- פרסום רכב, גיל מינימלי, מסירה, עלות מסירה, אזור ציבורי וכתובת פרטית.
- תמונה אוטומטית לרכב כפתרון fallback.
- הזמנות, אישור/דחייה/ביטול/סיום עם מעבר סטטוס מבוקר.
- הודעות לפי הזמנה.
- הוכחת תשלום עם תמונה וסכום.
- תיעוד לפני נסיעה והחזרה: סרטון חוץ, תמונת דלק/מיילים, מספר מיילים, דלק והערות.
- מנהל רואה משתמשים ומאשר/דוחה אימות.
- בעל רכב רואה פרטי שוכר ומסמכים בקישור זמני רק בזמן הרלוונטי.

## בדיקות שבוצעו בסביבה זו
- `node --check` לכל מודולי JavaScript, Service Worker ו-Netlify Functions.
- בדיקה סטטית שמכשילה build אם חוזרת כתיבת state מלאה או מזהי HTML כפולים בקובץ הראשי.
- Unit tests ל-HTML escaping, מדיניות סיסמה ומיפוי סטטוסים.
- ZIP integrity.

## מה עדיין חייב אימות ב-Staging
אין אפשרות אמינה לטעון מכאן את Firebase/Netlify הפרטיים שלך עם שלושה חשבונות אמיתיים. לכן לפני חי חובה לבדוק ב-Staging:
1. הרשמה, כניסה, יציאה ורענון רגיל/בסתר.
2. שלושה חשבונות: מנהל, בעל רכב ושוכר.
3. העלאה וצפייה במסמכים לאחר הגדרת משתני השרת.
4. מעבר מלא של הזמנה עד סיום.
5. ניסיון גישה לא מורשה למסמך ולכתובת פרטית — חייב להחזיר 403.
6. העתקת legacy ובדיקת ספירות מול הגיבוי.

## הערה
זו בנייה מחדש, לא תיקון קוסמטי של הקובץ הישן. יש לפרוס קודם ב-Staging, לפרסם Rules V2 ולהגדיר את משתני Netlify המתוארים ב-`SETUP_V2_HE.md`.
