פתרון משתמשים חדשים באתר CrownDrive

אין צורך להכניס מיילים ידנית ב-Firebase.
כל משתמש חדש יכול להירשם לבד באתר עם המייל והסיסמה שלו.

מה עודכן בגרסה הזאת:
1. משתמש חדש שלוחץ בטעות על כניסה עם מייל שעדיין לא קיים — האתר מנסה ליצור לו חשבון Firebase Auth אוטומטית.
2. אם המייל כבר קיים אבל הסיסמה שגויה — האתר מציג הודעה ברורה ולא יוצר חשבון כפול.
3. משתמשים חדשים נכתבים ל-Firebase Authentication ול-Realtime Database.
4. המשתמשים עדיין חייבים להיות מחוברים כדי לכתוב לדאטאבייס, כך שה-Rules עם auth != null ממשיכים לעבוד.

Firebase דרוש:
Authentication -> Sign-in method -> Email/Password -> Enabled
Realtime Database Rules:
{
  "rules": {
    "crowndrive-live": {
      ".read": true,
      ".write": "auth != null"
    },
    "$other": {
      ".read": false,
      ".write": false
    }
  }
}
