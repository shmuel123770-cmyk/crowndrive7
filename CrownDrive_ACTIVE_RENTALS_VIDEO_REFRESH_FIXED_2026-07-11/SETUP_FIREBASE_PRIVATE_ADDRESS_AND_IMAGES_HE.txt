הפעלה — כתובות פרטיות ותמונות ב-CrownDrive

1. Firebase Authentication
- ודא שספק Email/Password פעיל.
- סיסמאות נשמרות רק ב-Firebase Authentication; האתר אינו שומר pass ב-State או localStorage.

2. Realtime Database Rules
- Firebase Console > Realtime Database > Rules.
- העתק את כל התוכן של FIREBASE_DATABASE_RULES_PRIVATE_ADDRESS.txt ולחץ Publish.
- תחת crowndrive-live/admins צור ידנית רשומה שבה המפתח הוא Firebase Auth UID של המנהל והערך true.
  דוגמה: crowndrive-live/admins/ABC123 = true
- בלי UID מנהל, המנהל לא יוכל להעביר כתובות ישנות לנתיב הפרטי או לערוך כתובת של בעל רכב אחר.

3. Firebase Storage
- Firebase Console > Storage > Get started.
- בחר אזור והשלם יצירת bucket.
- Storage > Rules: העתק את FIREBASE_STORAGE_RULES.txt ולחץ Publish.
- התמונות והחתימות נשמרות ב-Storage. ב-Realtime Database נשמר רק URL.

4. מבנה הכתובות
- מידע ציבורי נשאר בתוך crowndrive-live/state/data/cars ללא fullAddress.
- כתובת מלאה נשמרת ב-crowndrive-live/privateCarDetails/{carId}.
- לאחר אישור הזמנה נוצרת הרשאה ב-crowndrive-live/approvedCarAccess/{carId}/{renterUid}=true.
- רק בעל הרכב, מנהל שהוגדר ב-admins, או שוכר שקיבל הרשאה יכולים לקרוא כתובת מלאה.

5. מעבר מנתונים ישנים
- היכנס פעם אחת כבעל כל רכב, או כמנהל שה-UID שלו הוגדר תחת admins.
- האתר יעביר כתובות ישנות מתוך cars ל-privateCarDetails וימחק אותן מה-State הציבורי בכתיבה הבאה.
- לפני המעבר מומלץ לבצע Export JSON של Realtime Database לגיבוי.

6. פריסה
- העלה את תוכן ה-ZIP ל-main.
- Netlify > Deploys > Trigger deploy > Deploy project without cache.
