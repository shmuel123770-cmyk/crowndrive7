CrownDrive — FINAL build (2026-07-12, rev.11)
Original CrownDrive design (Heebo, steel-blue #3F6B8E, dark #1B222B) on the secure V2 backend.

rev.11 (2026-07-12): server upload made reliable + self-diagnosing. media-upload now writes via
the Firebase Storage REST API using the Admin credential's OAuth token (built-in fetch, no
@google-cloud/storage bundling to fail on Netlify); auto-creates the download token. Bucket name
is derived/normalised (handles a wrong .appspot.com env). Added @google-cloud/storage to
external_node_modules (for the other storage functions). If a write still fails, the Hebrew error
now names the cause (Storage IAM permission / bucket / missing component / service-account key).
Prices confirmed all USD ($) via money(). Assets unchanged (?v=13).

rev.10 (2026-07-12): any image type + any source size. Server (media-upload) accepts any
image/* type; client re-encodes to a high-quality JPEG at up to 2560px, stepping quality (and
if needed dimensions) down so even a worst-case source stays under the request limit. So a photo
of any type and any size uploads successfully, auto-optimized for quality + fast loading.
(Assets unchanged at ?v=13 — only network-first modules + the function changed.)

rev.9 (2026-07-12) — DEFINITIVE upload fix:
- Confirmed via the live site that direct client→Firebase-Storage uploads simply do not work in
  in-app browsers (Telegram/IG webviews block the cross-origin request). New approach: images are
  resized client-side (<=1600px JPEG) and POSTed as base64 to a NEW same-origin function
  (media-upload.mjs) that writes them with the Admin SDK and returns a permanent token URL.
  This works in every browser and needs NO Storage rules (Admin SDK bypasses them). Videos keep
  the SDK path (too large for a base64 body). Assets ?v=13; 19 server functions; 76 scenarios pass.

rev.8 (2026-07-12):

  a dedicated admin sign-in screen (email+password only, no renter/owner step). openAdminLogin().
- Maintenance mode now shows two interlocking rotating gears (steel-blue + gold) instead of a

rev.7 (2026-07-12):
- FIX upload + the "Unexpected … JSON at position 6" toast: getDownloadURL() made a second
  network request that in-app browsers (Telegram/Instagram webviews) corrupt, throwing a raw
  JSON error that was toasted and blocked the photo. Now the display url is built from the
  token in the upload response itself — no second request — with getDownloadURL only as fallback.
- Manufacturer images: now pulled from the Wikipedia article's lead image (a clean, current-
  generation photo) instead of random Commons uploads (no more "dirty Moscow Jeep"); Commons is
  a filtered fallback that rejects wash/dirty/snow/damaged/old-year shots and prefers recent years.
- Daily & hourly price are now optional; a car with no daily price shows "מחיר לפי בקשה".
- All user-facing errors are Hebrew only: the global handler now swallows any non-Hebrew /
  technical message (shows nothing) and never leaks SDK/JSON English text.
- Fixed admin verificationStatus subscription (added a top-level admin read rule) — was
  permission-denied, flooding the console and breaking the admin verification display. Assets ?v=11.

rev.6 (2026-07-12):
- FIX "stuck on complete-profile": accounts with a name but no role could never enter their
  area (the client rule blocks writing a role to an existing profile). Now the role is set

- Personal area made more professional: redesigned dashboard header (personal greeting +

  hid the redundant top-bar actions while inside the personal area. Assets bumped to ?v=10.

rev.4 fixes (2026-07-12):

  (firebasestorage.googleapis.com — CORS-enabled by Google) instead of a raw PUT to a
  signed storage.googleapis.com URL, which needed bucket CORS that can't be set from the

- JSON error toast ("תו בלתי צפוי … JSON במיקום 6") eliminated: every fetch/parse is guarded
  and the global handler suppresses raw SyntaxError text.

  rule) with a server fallback, so it always works; the same button turns the site back to
  normal ("האתר בתחזוקה — לחצו לפתיחה"). Requires the updated Realtime DB rules (config branch).
- Faster load + robust cache: service worker now cache-first for versioned assets, network-first
  for HTML/modules (never serves a stale/mismatched bundle); JS modules preloaded; assets ?v=9.
- 12,500-iteration fuzz suite added (tests/stress.test.mjs); all checks + unit tests pass.
