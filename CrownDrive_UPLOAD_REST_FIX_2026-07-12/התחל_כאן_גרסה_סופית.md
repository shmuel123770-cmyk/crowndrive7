## מסלול קצר לעלייה לאוויר (5 צעדים)

### 1) משתני סביבה ב-Netlify
Project configuration → Environment variables → הוסף שלושה:
| שם | ערך |
|----|-----|
| `FIREBASE_SERVICE_ACCOUNT_JSON` | תוכן ה-JSON המלא של Service Account (סוד — לא להעלות ל-Git) |
| `FIREBASE_DATABASE_URL` | `https://amar-75684-default-rtdb.firebaseio.com` |
| `FIREBASE_STORAGE_BUCKET` | `amar-75684.firebasestorage.app` |

**להשגת ה-Service Account:** Firebase Console → Project settings → Service accounts → Generate new private key → העתק את כל תוכן הקובץ אל המשתנה, ואז מחק את הקובץ מהמחשב.

### 2) כללי Realtime Database
Firebase → Realtime Database → Rules → הדבק את כל התוכן של `FIREBASE_DATABASE_RULES_V2.json` → Publish.

### 3) כללי Storage
Firebase → Storage → Rules → הדבק את `FIREBASE_STORAGE_RULES_V2.txt` → Publish.
ואז הרץ ב-Google Cloud Shell (מעדכן CORS להעלאות חתומות):
```bash
gcloud storage buckets update gs://amar-75684.firebasestorage.app --cors-file=storage-cors.json
```
(קובץ ה-CORS כבר כולל את `crowndrive7.netlify.app`. אם הדומיין שלך שונה — הוסף אותו לקובץ קודם.)

### 4) הגדר את עצמך כמנהל
ב-Realtime Database צור צומת:
```
admins/<ה-UID-שלך-מ-Firebase-Auth> = true      (ערך Boolean אמיתי, לא מחרוזת)
```

### 5) פרוס את **הגרסה הזו** (חשוב!)
הפריסה שנכשלה קודם רצה מקוד ישן ללא התיקון. ודא שאתה פורס את הקבצים מהחבילה הזו:
- **הכי פשוט:** Netlify → Deploys → גרור את התיקייה לאזור drag-and-drop.
- **דרך Git:** החלף את כל תוכן ה-repo בקבצים האלה, commit + push.

## למה הפריסה נכשלה קודם (ואיך זה מתוקן)
esbuild (ה-bundler של Netlify) לא מסוגל לצרור את `firebase-admin` ואת שרשרת התלויות שלו (`google-gax`, `google-auth-library`, `gaxios`, `gtoken`, `retry-request`, `@firebase/database-compat`), כי הן משתמשות ב-require דינמי. **הוספת החבילות ל-package.json לא פותרת** — הן כבר מותקנות. הפתרון הוא לסמן אותן `external` כדי שיישלחו כמודולים אמיתיים בתוך הפונקציה. זה כבר מוגדר ב-`netlify.toml`:
```toml
[functions]
  node_bundler = "esbuild"
  external_node_modules = ["firebase-admin", "@firebase/database-compat", ...]
```
