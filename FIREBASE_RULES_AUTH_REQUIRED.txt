חוקי Firebase Realtime Database — גרסה מאובטחת (חובה לעדכן!)
================================================================

⚠️ חשוב: החוקים הישנים היו ".read": true — כלומר כל אדם באינטרנט יכול היה
לקרוא את כל נתוני האתר (משתמשים, טלפונים, צילומי רישיון, צ'אטים) בלי להתחבר.

יש להדביק את החוקים האלה ב:
Firebase Console → Realtime Database → Rules → Publish

{
  "rules": {
    "crowndrive-live": {
      ".read": "auth != null",
      ".write": "auth != null"
    },
    "$other": {
      ".read": false,
      ".write": false
    }
  }
}

מה זה נותן:
- קריאה וכתיבה רק למשתמשים מחוברים (Firebase Authentication).
- האתר כבר מחתים כל משתמש ל-Firebase Auth אוטומטית בכניסה/הרשמה, אז שום דבר לא נשבר.

מה עוד תוקן בקוד:
- סיסמאות (pass) כבר לא נשלחות ל-Firebase בכלל — נשארות רק במכשיר של המשתמש.
- סיסמת המנהל הוסרה מקוד המקור של האתר.

מגבלה שנשארה (חשוב לדעת):
- כל המידע יושב ב-node משותף אחד, ולכן כל משתמש רשום יכול טכנית לקרוא/לשנות
  את הנתונים המשותפים (כולל צילומי רישיון שצורפו להזמנות). כדי לסגור את זה לגמרי
  צריך לפצל את המבנה ל-nodes לפי משתמש עם חוקים פרטניים — שינוי ארכיטקטוני גדול יותר.
