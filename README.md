# CrownDrive V2

בנייה מחדש מודולרית של CrownDrive, שנועדה להחליף את קובץ ה־HTML הענק והלא יציב.

## מה תוקן
- מאזין Firebase Auth יחיד.
- אין `RT_REF.set` ואין כתיבת state מלאה.
- אין מסד Netlify זמני בזיכרון.
- כל כתיבה עסקית עוברת דרך Netlify Functions עם אימות והרשאות.
- נתיבים נפרדים למשתמשים, רכבים, הזמנות, הודעות, תשלומים ודירוגים.
- רישיונות, סלפי, סרטונים והוכחות תשלום נשמרים ב־Storage פרטי ונפתחים באמצעות Signed URLs קצרים.
- בעל רכב רואה מסמכי שוכר רק בהזמנה פתוחה; לאחר סיום/דחייה הגישה נעלמת.
- כתובת מלאה נחשפת לשוכר רק לאחר אישור ההזמנה.
- מניעת הזמנות חופפות, אימות גיל, אימות מייל ומסמכים.
- תיעוד לפני/אחרי, הוכחת תשלום, הודעות ודירוגים.
- כלי העברה שאינו מוחק את המבנה הישן.
- בדיקות תחביר, אבטחה ו־unit tests כחלק מה־Deploy.

## מבנה
- `js/` — קוד דפדפן מודולרי
- `netlify/functions/` — פעולות שרת מאובטחות
- `css/app.css` — שכבת עיצוב אחת
- `tests/` — בדיקות סטטיות ויחידה
- `FIREBASE_DATABASE_RULES_V2.json` — כללי מסד
- `FIREBASE_STORAGE_RULES_V2.txt` — כללי אחסון
- `SETUP_V2_HE.md` — מדריך פריסה מלא

## בדיקות
```bash
npm install
npm run build
```

בדיקת Chromium עשויה להידלג בסביבות sandbox שאינן מאפשרות לפתוח דפדפן. הבדיקות הסטטיות והיחידה עדיין רצות וחוסמות פריסה במקרה של כשל.
