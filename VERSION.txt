CrownDrive — FINAL build (2026-07-12, rev.5)
Original CrownDrive design (Heebo, steel-blue #3F6B8E, dark #1B222B) on the secure V2 backend.

rev.5 (2026-07-12):
- Admin area now has a "צ׳אטים" tab (opens the full messaging page) and a profile view with
  avatar upload (via the header avatar) — admin can set a profile picture like everyone else.
- Chat is fully real-time: the admin thread list now live-updates/re-orders on every incoming
  message (was a one-time fetch); open conversations were already live.
- Car photos: removed the 2-photo minimum — 1 to 6 photos allowed (at least one).
- Manufacturer image search rewritten: scores by exact make+model(+year) and prefers official
  exterior/press shots, so it returns the right car or nothing (never a wrong look-alike).
- More admin control: promote/demote a user's role (owner↔renter) from the user modal
  (adds to existing block/delete/edit, car edit/hide/reassign/delete, booking edit, maintenance).

rev.4 fixes (2026-07-12):
- PHOTO/VIDEO UPLOAD FIXED: uploads now go through the Firebase Storage SDK
  (firebasestorage.googleapis.com — CORS-enabled by Google) instead of a raw PUT to a
  signed storage.googleapis.com URL, which needed bucket CORS that can't be set from the
  Firebase console. Requires the updated Storage rules (already in FIREBASE_STORAGE_RULES_V2.txt).
- JSON error toast ("תו בלתי צפוי … JSON במיקום 6") eliminated: every fetch/parse is guarded
  and the global handler suppresses raw SyntaxError text.
- MAINTENANCE MODE FIXED: the admin toggle writes config/maintenance directly (admin-only DB
  rule) with a server fallback, so it always works; the same button turns the site back to
  normal ("האתר בתחזוקה — לחצו לפתיחה"). Requires the updated Realtime DB rules (config branch).
- Faster load + robust cache: service worker now cache-first for versioned assets, network-first
  for HTML/modules (never serves a stale/mismatched bundle); JS modules preloaded; assets ?v=9.
- 12,500-iteration fuzz suite added (tests/stress.test.mjs); all checks + unit tests pass.

Home: hero + animated car, booking widget, owner CTA, up to 8 available cars + "לכל הרכבים",
"איך זה עובד" (3 steps), "אמון ובטיחות" (4 trust cards).

Cars page: filter bar (search / availability / type / fuel / gear / sort). Car cards: photo,
availability badge (פנוי/מושכר), make·model·year·type·fuel·gear, rating, price, owner name,
video badge. Detail modal: photo gallery + thumbnails + optional demo video + reviews + booking.

Add-car form: make→model→year from lists (with "אחר" free text), sub-model optional, car type,
2–6 photos with a chosen main photo, optional short video, manufacturer image auto-fetch.
Server (car-action) stores category/fuel/gear/seats/photos[]/videoUrl/ownerName; requires ≥2 photos.

Chats: dedicated page, WhatsApp-style thread list (right) + conversation (left), mobile slide.
Chat is open only from booking approval until the rental ends. Admin gets a full users list and
can message any user (messages/admin/<uid>). Before a rental starts the renter must send through
the chat: exterior video + fuel photo + odometer photo + payment proof; the owner then has two
buttons — "התחלת השכרה" (gated on those four) and "סיום השכרה".

Unified dashboards (admin/owner/renter). Deploy fixes retained (firebase-admin external, no-op
build, CSP fonts). New server function: car-media-public (makes owner car media public).
All static/security checks and unit tests pass; 17 Netlify functions. Firebase project: amar-75684.
See התחל_כאן_גרסה_סופית.md to deploy. Remember to publish the updated Realtime DB rules
